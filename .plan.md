# 物流状态 + 全局UI美化 实施方案

## 目标
1. 将物流状态持久化到数据库，在记录列表中直接显示
2. 全局美化前端 UI，提升视觉体验

---

## 一、数据库改动

### records 表新增字段
```sql
ALTER TABLE records ADD COLUMN tracking_state TEXT NOT NULL DEFAULT '';
ALTER TABLE records ADD COLUMN tracking_state_text TEXT NOT NULL DEFAULT '未查询';
ALTER TABLE records ADD COLUMN tracking_latest_time TEXT NOT NULL DEFAULT '';
ALTER TABLE records ADD COLUMN tracking_latest_context TEXT NOT NULL DEFAULT '';
ALTER TABLE records ADD COLUMN tracking_updated_at TEXT;
```

- `tracking_state`: 快递100返回的状态码（0-6）
- `tracking_state_text`: 中文状态（运输中/已签收/问题件等）
- `tracking_latest_time`: 最新物流时间
- `tracking_latest_context`: 最新物流描述
- `tracking_updated_at`: 上次查询时间

不新建 tracking_events 表 —— 物流详情通过实时查询快递100获取，数据库只缓存最新状态用于列表展示。

---

## 二、后端改动

### database.py
- `init_db()` 中用 `ALTER TABLE ... ADD COLUMN` 迁移新增字段（IF NOT EXISTS 包裹）

### models.py
- RecordCreate / RecordUpdate 中加入 tracking_state 等新字段（可选）

### routers/tracking.py
新增两个端点：
- `POST /api/tracking/sync` — 批量刷新所有有快递单号的记录的物流状态，写入 records 表
- `GET /api/tracking/status/{record_id}` — 查询单条记录的物流详情（实时查快递100 + 返回完整历史）

### routers/records.py
- list_records 返回新字段

## 三、前端 UI 美化

### 整体风格：毛玻璃 + 渐变（用户已确认）
- Header: 渐变紫蓝背景 + backdrop-blur 毛玻璃
- 卡片: 白色半透明 + 模糊阴影
- 按钮: 渐变色 + hover 发光效果
- 表格: 斑马纹 + 柔和 hover

### 具体改动

#### Header
- 渐变背景 + 毛玻璃效果
- 更大气的标题样式

#### 仪表盘
- 统计卡片加入图标 + 渐变背景
- 添加物流状态概览卡片（运输中/已签收/问题件数量）

#### 记录列表表格
- 新增「物流状态」列，用彩色 badge 显示（运输中=蓝色、已签收=绿色、问题件=红色）
- 行 hover 效果更柔和
- 表头样式优化

#### 快递查询页
- 改为从记录列表中一键同步物流状态
- 保留手动查询功能，但优化展示

#### 新增/编辑弹窗
- 更精致的表单样式
- 分组布局（基本信息 / 财务信息 / 物流信息）

#### 全局
- 更好的字体层级
- 按钮加入微动画
- 加载状态优化（骨架屏或 spinner）
- 响应式优化
